先看一个图：

对于第一种插入方式：head作为一个ListNode的引用，他在进程栈中存放的是一个地址（假设是0x0010），执行temp=head时，temp会被赋值（0x0010），若此时执行的判断时temp!=null，那么最后会运行temp=null,此时temp中存放的不是任何一个地址，而是null。这就好比，temp此时和head单链表没有一点关系了，所以你后面的new ListNode()也就没有意义了。
对于第二种插入方式：前面都是一样的，不同的是判断了temp.next!=null，那么执行完while循环后，temp存放的是head单链表的尾节点的地址值，所以执行temp.next=new ListNode()之后，是能够将数据插进去的。

Remove Nth Node From End of List

发表于 2016-12-29 | | 阅读次数

脑子僵化了，怎么也想不出遍历一次的算法，总以为要先知道链表长度，才能准确定位到要skip的节点，后来才知道利用双指针可以很简单的解决这个问题。原理：删去倒数第n个节点，这个时候，被删的节点（我们假设是a）和链表的末尾（空节点）距离是知道的（距离为n），我们只要定义两个指针p1、p2，指针p2先移动n+1个距离。有人会问为什么是n+1个距离不是n，因为我们不是为了找到那个被删的节点，而是要删掉到，即跳过该节点。接下来就是同时移动指针p1、p2，直到p2指向链表末尾（指向null）。这个时候p1的位置就是要被删除的节点的前一个位置，最后只用p1.next=p1.next.next;这样就删除了这个节点。

第三章 Gradle的生命周期

Gradle生命周期

包括三个阶段：initialization、configuration和execution。
Initialization：这个时期，Gradle会决定哪些project参与到build中来，并为每一个project创建实例对象。
Configuration：在这个阶段配置每个Project的实例对象，比如将task表示成Gradle自己的model方便管理和后续操作。然后就执行这些项目脚本中的一部分task。
Execution：Gradle会执行那些在配置阶段已经创建和配置好的task子集，Gradle会将这些子集的名字作为参数传递给Gradle命令和当前目录。

Settings file设置文件

Gradle定义了一个设置文件，默认名称为settings.gradle在Gradle的初始化阶段(Initialization)加载settings.gradle文件。
多项目的工程settings.gradle文件在工程的根目录。
对于单个项目settings.gradle是可选择的，但是多个项目的工程它是必须要有的。
Settings.gradle定义了工程中包含的项目，你也有可能会添加一些库文件到脚本路径中。
对于一个构建脚本，属性和方法的调用被委托给一个Project类的实例对象。所以类似的settings.gradle中属性和方法被委托给一个Settings类的实例对象。
如果从settings.gradlea角度来看，initialization阶段做得就是找到settins.gradle并解析他，判断是多项目还是单项目。

第二章 Gradle中task的简单使用

发表于 2016-12-27 | | 阅读次数

每一个Gradle构建都包含三个基本构建块：project、task和property。其中一个project表示一个正在构建的组件比如JAR或者一个正在完成的任务，如部署应用程序。当构建进程启动后，Gradle会基于build.gradle中的配置实例化project类，同样的task也是会实例化对应的DefaultTask，Gradle向我们隐藏了实例化的过程，但是暴露了一系列的API。一个项目对应一个build.gradle文件，由于刚开始学习的时候是单项目，所以暂时不用考虑其他的。

Project的相关API

apply(options:Map):表示应用哪一种插件，一般有java、android等。
buildScript(config:Closure)：构建的配置
dependencies：依赖管理
getter/setter属性
file：创建文件
task：创建任务

Task

一个project中可以包含1到n个task，task之间可以相互依赖，其中被依赖的那个task要首先执行。

扩展属性

要用ext命名空间来添加自定义的属性，但是对于已有的属性，比如poject中的version、description等可以通过setter/getter来访问。

DefaultTask

上面说股过task是DefaultTask的类型，会被实例化，然后通过setter/getter来访问task下的所有私有属性，由于Groovy为Gradle提供了糖衣，所以可以直接使用属性名来访问属性，他在底层调用了setter/getter方法。DefaultTask提供了group表示该task属于哪一组任务组，提供了description表示该任务的描述。

Letter Combinations of a Phone Number

发表于 2016-12-27 | | 阅读次数

其实题目的意思比较好理解，但是做了一会之后，发现是个组合问题，要进行多少次循环是根据digits总的长度和每一个数字对应的字母个数有关。貌似这个循环不好写，因为循环个数不是固定和可以看作固定的（比如100次，n*n次等等这种类型的），但是我们可以发现，每一次做得循环是固定的，都是那个数字代表的字母个数，比如当前处理的是“6”，那么当前要循环3次，若是“7”则要循环4次，且没处理完一个数字后，得到的中间结果个数要乘上3或4，这样一来就好办了，我们只用将当前的数字代表的字母和中间结果进行组合，再用组合得到的结果去替代中间结果，问题解决。

Gradle简介

Gradle是一个自动化构建工具，用来帮助我们更好地管理项目，java中叫构建build，c/c++貌似叫make，其实不管叫什么，本质都是一样的。Gradle、Maven、Ant被称为是java世界中的三大构建工具，前面两种我就不介绍了，他们都是基于xml的构建工具，在逻辑处理和代码简洁等其他方面远不如Gradle。

那么，Gradle到底在构建的时候能帮助我们做哪些事呢？大家一定有用记事本写java的经历，写完java文件后，要在cmd窗口下使用javac和java命令这样会生成一个class的文件，并运行。这只是一个很简单的java编译运行的过程，如果手动的调用jdk的话，非常麻烦，但是在eclipse的IDE中，这些编译、测试以及依赖管理等其他功能都帮你完成了，这个IDE就是借助了自动化构建工具完成的，它向程序员提供的仅仅是run的按钮，轻轻一点，IDE会自动为你完成这些功能。
