Minimum Path Sum

发表于 2017-02-08 | | 阅读次数

由于，限制了每一步只能down或者right，所以最后一行和最后一列的位置的MinimumPathSum一定是当前位置的值加上前一个位置的MinimumPathSum。为了方便计算，我们可以从[0][0]开始到[m-1][n-1],并且第一个位置的MinimumPathSum的值是知道的（即grid[0][0]）。然后依次计算第一行和第一列的值，最后计算剩下位置的MinimumPathSum。

Unique Paths2

发表于 2017-02-01 | | 阅读次数

Unique Paths的变体，主要是增加了一个障碍物的考虑。因此，在使用打表法的时候，要对当前位置进行判断，如果是障碍物则将该位置的paths数目置为0。另外，再对行数或列数为1的位置，要进行迭代，即每一个位置的paths数目都是依赖于前一个位置。以行数为例，[0][i]的数目是等于[0][i-1]的paths数目的，因此只需要将[0][0]的值提前给出，依据这个值将其他位置的值求出即可。

Unique Paths

发表于 2017-01-31 | | 阅读次数

代码部分十分简单，一开始只想到用DFS方法，后来提交发现超时了。
最后想了一会，每一个位置到end的路径数量=uniquePaths(m-1,n)+uniquePaths(m,n-1)。可以知道当n=1或者m=1时，路径数为1.所以可以利用打表法，将mn位置的所有点的uniquePaths都能算出来。时间复杂度为mn。
