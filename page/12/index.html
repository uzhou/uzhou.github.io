第二章 Gradle中task的简单使用

发表于 2016-12-27 | | 阅读次数

每一个Gradle构建都包含三个基本构建块：project、task和property。其中一个project表示一个正在构建的组件比如JAR或者一个正在完成的任务，如部署应用程序。当构建进程启动后，Gradle会基于build.gradle中的配置实例化project类，同样的task也是会实例化对应的DefaultTask，Gradle向我们隐藏了实例化的过程，但是暴露了一系列的API。一个项目对应一个build.gradle文件，由于刚开始学习的时候是单项目，所以暂时不用考虑其他的。

Project的相关API

apply(options:Map):表示应用哪一种插件，一般有java、android等。
buildScript(config:Closure)：构建的配置
dependencies：依赖管理
getter/setter属性
file：创建文件
task：创建任务

Task

一个project中可以包含1到n个task，task之间可以相互依赖，其中被依赖的那个task要首先执行。

扩展属性

要用ext命名空间来添加自定义的属性，但是对于已有的属性，比如poject中的version、description等可以通过setter/getter来访问。

DefaultTask

上面说股过task是DefaultTask的类型，会被实例化，然后通过setter/getter来访问task下的所有私有属性，由于Groovy为Gradle提供了糖衣，所以可以直接使用属性名来访问属性，他在底层调用了setter/getter方法。DefaultTask提供了group表示该task属于哪一组任务组，提供了description表示该任务的描述。

Letter Combinations of a Phone Number

其实题目的意思比较好理解，但是做了一会之后，发现是个组合问题，要进行多少次循环是根据digits总的长度和每一个数字对应的字母个数有关。貌似这个循环不好写，因为循环个数不是固定和可以看作固定的（比如100次，n*n次等等这种类型的），但是我们可以发现，每一次做得循环是固定的，都是那个数字代表的字母个数，比如当前处理的是“6”，那么当前要循环3次，若是“7”则要循环4次，且没处理完一个数字后，得到的中间结果个数要乘上3或4，这样一来就好办了，我们只用将当前的数字代表的字母和中间结果进行组合，再用组合得到的结果去替代中间结果，问题解决。

Gradle简介

Gradle是一个自动化构建工具，用来帮助我们更好地管理项目，java中叫构建build，c/c++貌似叫make，其实不管叫什么，本质都是一样的。Gradle、Maven、Ant被称为是java世界中的三大构建工具，前面两种我就不介绍了，他们都是基于xml的构建工具，在逻辑处理和代码简洁等其他方面远不如Gradle。

那么，Gradle到底在构建的时候能帮助我们做哪些事呢？大家一定有用记事本写java的经历，写完java文件后，要在cmd窗口下使用javac和java命令这样会生成一个class的文件，并运行。这只是一个很简单的java编译运行的过程，如果手动的调用jdk的话，非常麻烦，但是在eclipse的IDE中，这些编译、测试以及依赖管理等其他功能都帮你完成了，这个IDE就是借助了自动化构建工具完成的，它向程序员提供的仅仅是run的按钮，轻轻一点，IDE会自动为你完成这些功能。

Linux中的内存管理

发表于 2016-12-23 | | 阅读次数

一、物理内存和虚拟内存

物理内存就是系统硬件提供的内存大小，是真正的内存，相对于物理内存，在linux下还有一个虚拟内存的概念，虚拟内存就是为了满足物理内存的不足而提出的策略，它是利用磁盘空间虚拟出的一块逻辑内存，用作虚拟内存的磁盘空间被称为交换空间（Swap Space）。

作为物理内存的扩展，linux会在物理内存不足时，使用交换分区的虚拟内存，更详细的说，就是内核会将暂时不用的内存块信息写到交换空间，这样以来，物理内存得到了释放，这块内存就可以用于其它目的，当需要用到原始的内容时，这些信息会被重新从交换空间读入物理内存。

Linux的内存管理采取的是分页存取机制，为了保证物理内存能得到充分的利用，内核会在适当的时候将物理内存中不经常使用的数据块自动交换到虚拟内存中，而将经常使用的信息保留到物理内存。

要深入了解linux内存运行机制，需要知道下面提到的几个方面：

Linux系统会不时的进行页面交换操作，以保持尽可能多的空闲物理内存，即使并没有什么事情需要内存，Linux也会交换出暂时不用的内存页面。这可以避免等待交换所需的时间。
Linux 进行页面交换是有条件的，不是所有页面在不用时都交换到虚拟内存，linux内核根据”最近最经常使用“算法，仅仅将一些不经常使用的页面文件交换到虚拟 内存，有时我们会看到这么一个现象：linux物理内存还有很多，但是交换空间也使用了很多。其实，这并不奇怪，例如，一个占用很大内存的进程运行时，需 要耗费很多内存资源，此时就会有一些不常用页面文件被交换到虚拟内存中，但后来这个占用很多内存资源的进程结束并释放了很多内存时，刚才被交换出去的页面 文件并不会自动的交换进物理内存，除非有这个必要，那么此刻系统物理内存就会空闲很多，同时交换空间也在被使用，就出现了刚才所说的现象了。关于这点，不 用担心什么，只要知道是怎么一回事就可以了。
交换空间的页面在使用时会首先被交换到物理内存，如果此时没有足够的物理内存来容纳这些页 面，它们又会被马上交换出去，如此以来，虚拟内存中可能没有足够空间来存储这些交换页面，最终会导致linux出现假死机、服务异常等问题，linux虽 然可以在一段时间内自行恢复，但是恢复后的系统已经基本不可用了。

linux中断向量是如何分配的

发表于 2016-12-23 | | 阅读次数
中断的概念

所谓中断，是指CPU在正常运行程序时，由于程序的预先安排或内外部事件，引起CPU中断正在运行的程序，而转到发生中断事件程序中。这些引起程序中断的事件称为中断源。
其实从物理学的角度看，中断是一种电信号，由硬件设备产生，并直接送入中断控制器（如 8259A）的输入引脚上，然后再由中断控制器向处理器发送相应的信号。处理器一经检测到该信号，便中断自己当前正在处理的工作，转而去处理中断。此后，处理器会通知 OS 已经产生中断。这样，OS 就可以对这个中断进行适当的处理。不同的设备对应的中断不同，而每个中断都通过一个唯一的数字标识，这些值通常被称为中断请求线。

中断的分配

编号0～31的向量对应于异常和非屏蔽中断（这类也属于向量中断或者内部中断，向量中断的特性就是一一映射，一个中断向量对应一个中断处理例程）

编号32～47的向量（即由IO设备引起的中断）分配给屏蔽中断。（这类也属于非向量中断或者外部中断，在Intel手册中常说的就是这类中断。非向量中断的特性在下文就可以体现出来。）

编号48～255的向量用来标示软中断。Linux用其中的128或0x80来实现系统调用（所以通过系统调用来进入内核空间都需要通过128中断操作）

非屏蔽中断的向量和异常的向量是固定的。

Linux中系统调用和函数调用

发表于 2016-12-21 | | 阅读次数
概念
首先，我们需要了解一下是什么是系统调用，什么是函数调用及两者之间的区别。


系统调用：是操作系统为用户态运行的进程和硬件设备(如CPU、磁盘、打印机等)进行交互提供的一组接口，即就是设置在应
用程序和硬件设备之间的一个接口层。可以说是操作系统留给用户程序的一个接口。再来说一下，linux内核是单内核，结构
紧凑，执行速度快，各个模块之间是直接调用的关系。放眼望整个linux系统，从上到下依次是用户进程->linux内核->硬件。
其中系统调用接口是位于Linux内核中的，如果再稍微细分一下的话，整个linux系统从上到下可以是：用户进程->系统调用
接口->linux内核子系统->硬件，也就是说Linux内核包括了系统调用接口和内核子系统两部分；或者从下到上可以是：物理硬
件->OS内核->OS服务->应用程序，其中操作系统起到“承上启下”的关键作用，向下管理物理硬件，向上为操作系服务和应用
程序提供接口，这里的接口就是系统调用了。
函数调用：其实就是对一些库函数的调用，顾名思义是把函
数放到库里。是把一些常用到的函数编完放到一个文件里，供别人用。别人用的时候把它所在的文件名用#include<>加到里面
就可以了。一般是放到lib文件里的。一般是指编译器提供的可在c源程序中调用的函数。可分为两类，一类是c语言标准规定
的库函数，一类是编译器特定的库函数。
系统调用和函数调用
进程运行需要内存空间,用户空间就是指应用程序的进程在内存中的执行位置 ,系统的核心空间是指包括一切系统核心程序代
码的内存空间。一般的应用程序的进程必须经过系统调用才能获得这些核心程序的服务.通常进程在运行时都是执行自己的代码
,此时称作用户进程, 但是,当这个进程调用了系统调用以后,那么它就进入了系统核心
空间 ,成为核心进程了,核心进程具有执行核心程序的权利。

就拿一个简单的copy程序来说，除了涉及到I/O操作，函数调用都是在用户空间内完成的，一般不会切换到和心态，系统调用
则是要求操作系统为用户的进程提供某种服务,通常是涉及系统的硬件资源和一些敏感的软件资源等。
函数库中的函数,尤其是与输入输出相关的函数,大多数必须过 LINUX 的系统调用来完成,因此,我们可以将函数库的函数当成是
应用程序设计人员与系统调用程序之间的一个中间层,通过这个中间层,我们可以用一致的接口来完全地调用系统调用.这样程
序设计人员可能只要写一次代码就能在不同版本LINUX间实现各种具体完全不同的系统调用。
总之,系统调用是在操作系统的核心态下执行的,而函数一般是在用户状态下执行的,某些时候,函数还要继续调用系统调用才能
完成特定的功能。
