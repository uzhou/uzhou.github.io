cifar10介绍

Cifar-10是由Hinton的两个大弟子Alex Krizhevsky、Ilya Sutskever收集的一个用于普适物体识别的数据集。其中包含了10中不同的物体，如airplane、cat、dog、deer等，cifar100则有100种物体分类。 cifar10由60000张32*32的RGB彩色图片构成，共10个分类。50000张训练，10000张测试（交叉验证）。具体介绍可以参考http://www.cnblogs.com/neopenx/p/4480701.html

Jump Game

发表于 2017-01-23 | | 阅读次数

这一题的关键在于0，如果能找到一个0不能被jump过去，那么整个游戏就失败了。而一个0能不能jump过去，则要看该位置之前的所有位置中能jump最远的位置是否大于该0的位置。整个算法的复杂度为n，其实该算法还可以优化，因为当出现连续的0的时候，只要该连续0的最后一个0能被jump过去那么整串0都能被jump过去，如果不能，则游戏失败。

Spiral Matrix

发表于 2017-01-23 | | 阅读次数

螺旋矩阵(m n)问题，如果是(n n)的矩阵那么十分简单。由于不是方阵，所以要考虑m和n的大小。要考虑的因素有：循环次数(受min(m,n)影响)；最后一个循环处理(受m,n大小影响)。一次性解决这个问题还是有点难度的，只能通过不断的提交找出自己的算法的不足，然后慢慢的修改就可以了。

Maximum Subarray

发表于 2017-01-23 | | 阅读次数

典型的动态规划，每次扫描一个数，都要判断是否对原有的状态(本题指max和sum)进行修改。并且要知道，被扫描的数不是判断原有状态是否需要修改的唯一条件，原有状态sum和max也是判断的条件之一。

Pow

发表于 2017-01-23 | | 阅读次数

实现pow函数，如果用暴力方法肯定会超时，所以需要找一个复杂度小于n的算法。复杂度主要由指数决定，指数n代表要将底数x乘n次，我们可以把n划分成1+2+4+…+2^k的形式，这样的复杂度就是logn。需要注意的是在循环的过程中，循环参数i有可能会在最后一次循环中超出Integer的范围，所以这里要做一个特判。另外还要对n是负数的情况进行处理，思想是转换成正数，然后取1/ans的值。

这一题主要是把具有相同字符的字符串分成一组，我们可以先将每一个string按照字符排序，若存在这个序列则将这个string放进该list，若没有则新建一个list并放进目标List> ans中。整体难度不大，不过要注意string和char数组之间的转换。

Rotate

发表于 2017-01-21 | | 阅读次数

矩阵顺时针旋转90度，要求是空间复杂度必须小于n*n，不能使用一个新的数组来解决问题。从几何的角度来看，矩阵顺时针旋转90度，也就代表了每一个元素也是围绕矩阵中心旋转。因此我们可以一层一层的去分解这个问题，每一层的旋转又可以看成是该层元素的前移。这样的话数学模型就建出来了，之后就是如何用代码实现。我给出的代码，有个地方(直接写成每次移动若干个位置)是可以去掉了。
