Search in Rotated Sorted Array

发表于 2017-03-07 | | 阅读次数

在一个反转的递增数组中进行查找的问题，如果是正常的递增序列，那么时间复杂度最快的是二分查找。很明显本题需要对二分查找略做修改，由于不存在重复元素，因此以该序列的中间位置mid作为分界。要么beginIndex到mid位置的元素序列为递增序列，要么从mid到endIndex位置的元素为递增序列。因此只要target在这个递增序列中，就可以进行二分查找，如果不在，那么说明在另一个序列中，此时只需要对另一个序列进行递归操作。

发表于 2017-03-06 | | 阅读次数

发现自己好长时间没有刷题了，惭愧！这种事落下一天就想落下第二天、第三天、、、，好在现在捡回来还不算太晚。
这一题是一个典型的递归问题，所以不算太难：

  1. 首先需要一个isMark的二维boolean数组，用于记录矩阵中的某个位置是否被遍历过；
  2. 确定递归过程中的起点状态、退出条件（包括超出边界，不满足题意等）、下一个递归的起点状态；
  3. 然后就是对每一个位置进行递归操作。
    上面说的也是典型的递归问题解决方法，一般的递归问题按照这种套路很容易就能解决。

Combinations

发表于 2017-02-13 | | 阅读次数

这一题是实质上是组合问题，C(n,k)。有点想到全排列的那几道题目，因此这一题我难免会往那个方向想。好在方向上可以，算法效率也不错，96.39%。具体思想是：

  1. 找出第一个组合：1、2、3、、、k;
  2. 从上一个组合的最后一个位置开始查找，找到第一个满足i<n-index的元素；（其中index是一个计数器，每当找不到这样的元素，该计数器+1）
  3. 若index==k，则说明所有的组合已经找到，返回-1；否则继续下一步；
  4. 第二步找到了第一个满足i<n-index的元素，我们从该元素开始，将后续元素的值依次改变。该元素值变为旧值+1，后续位置元素依次递加。

注意：如果使用递归，那么有可能会出现栈溢出的情况，所以最好要把递归问题改成非递归问题。

Climbing Stairs

发表于 2017-02-10 | | 阅读次数

这种题的第一想法就是暴力解决，使用DFS递归。但是已提交，发现超时，所以需要想另一种办法。类似的超时题很多，基本上到后来的解决方法都是牺牲空间并利用一定的规律完成的。这一题也是的，通过观察会发现，F[i]=F[i-1]+F[i-2]。这可以说是这一类题的套路了，以后遇到这种超时题，不妨往这个思路上想。
