对象与类

java中的思想就是万物皆对象，他是一个典型的OOP语言。OOP和面向过程编程的区别在于：OOP一般会先考虑数据结构，再去考虑算法；而面向过程编程则是先考虑算法，在考虑用什么数据结构。

Java学习笔记---String

发表于 2017-03-14 | | 阅读次数

String

String是Java标准库提供的一个预定义类，注意String并不是一个char类型的数组，或者说也不能将其等同于一个char类型数组。另外，String是final类型的，这就决定了他的众多性质。

  • 由于String是final型的，所以他的值是不可更改的。
  • 将String设为final型，主要是考虑其共享的效率，而不是考虑他拼接的效率。
  • 在进行字符串比较的时候，不能简单的将两个对象进行“==”操作，因为这样有可能正确，也有可能错误。解释：由于String会被编译器放进一个字符串常量的共享池内，但这并不是将所有的String都放进来，而只是共享定义的String变量，一些通过subString和“+”操作得到的String不会被共享进来，这样的话简单的“==”就不能判定是否两者String值相等，因此要使用equals函数。

StringBuilder

与String不同的是，StringBuilder更注于字符串的拼接，而忽略了共享的概念。不过StringBuilder并不是多线程安全的，他只是在单线程下安全。

StringBuffer

与StringBuilder不同的是，StringBuffer是多线程安全的。

Unique Binary Search Trees

第一想法是暴力法，枚举所有可能的序列，对此序列进行判断，看是否是合法的二叉搜索树，但后来感觉应该没有这么复杂。因为1~n的二叉搜索树个数等于k ~n+k的个数,所以一旦确定一个元素为根节点（假设该节点值为k，总节点数为n），那么其左边一定是小于该根节点元素的连续序列组成的二叉搜索树，个数为F[k-1],F[k-1]表示当总节点数为k-1时二叉搜索树的总个数；同理右边的个数为F[n-k]，总个数为两者相乘。

Partition List

分割链表操作，难点是保持原有元素的相对顺序不改变。如果另外分配空间的话十分简单，只需要重新建立一个链表就ok了。但是还是要想想有没有o（1）空间复杂度的算法。经过观察知道：大于等于val的数（有可能是连续的也有可能不是连续的）可以与后面的小于val的数交换。这样最后得到的链表就是要求的链表，时间复杂度为o（n）。
