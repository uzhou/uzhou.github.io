MongoDB入门--简介

发表于 2017-06-07 | | 阅读次数

概述

MongoDB 是一个基于分布式文件存储的数据库。由 C++ 语言编写。旨在为 WEB 应用提供可扩展的高性能数据存储解决方案。
MongoDB 是一个介于关系数据库和非关系数据库之间的产品，是非关系数据库当中功能最丰富，最像关系数据库的。

关系和非关系型数据库

既然MongoDB是非关系型数据，那么它和我们平时常用的关系型数据库比如MySQL有什么区别呢。
首先，这个“关系”代表的是什么？我们知道MySQL中对数据管理是通过表结构来管理的，一个表就代表了一个实体的集合，而关系就是实体与实体之间的关系（最常见的关系有1对1,1对多，多对多等）。
另外，菲关系型数据库遵循最终一致性原则，而不是关系型数据库的ACID原则。

元数据

元数据的定义比较简单，描述数据的数据。在MongoDB中，元数据指的是描述数据库信息的数据，这些数据都存放在system这个特殊集合中。

对MongoDB的第一印象

第一次接触MongoDB，最大的印象就是MongoDB中的数据存储是以BSON形式存储的，并且同一个集合中的不同的文档（相当于同一个table中不同的行）可以有不同的结构，但是一般文档之间的结构基本一致，这也许就是为什么说MongoDB是最像关系型数据库的非关系型数据库吧。

要实现单例模式必须有下面三个要点：

  1. 构造方法必须私有
  2. 要有一个私有的静态引用
  3. 因为构造方法私有，所以产生该单例对象的方法必须是static，该方法将为2中的私有引用赋值，如果已赋值则直接返回该引用。

Java设计模式--工厂模式

发表于 2017-05-31 | | 阅读次数

概述

工厂模式的目的就是为了将对象的创建从应用中隔离出来，从而降低耦合。主要体现了面向接口编程、依赖抽象而不是依赖具体类的原则。

简单工厂

工厂模式中，必然要涉及两个组件，一个是工厂组件，另一个则是需要创建的产品组件。我们按照对这两大组件的抽象程度可以将工厂模式分为简单工厂、工厂方法模式、抽象工厂模式。
简单工厂中，工厂组件是具体的，产品组件有抽象和实现的多个具体产品类。
这个工厂组件中一般有一个创建产品的具体方法（静态为主），该方法根据不同的需求返回不同的具体产品类对象。而凡是使用具体产品的业务逻辑只需要调用这个工厂对象的方法即可。
简单工厂模式将业务逻辑中可变的部分（根据不同的需求返回不同的产品对象）和不变的部分（对这些产品共有的操作）分离开来了。

工厂方法模式

工厂方法模式中，工厂组件有抽象的工厂和实现该抽象工厂的若干个工厂类，其中抽象工厂中包含一个创建产品对象的抽象方法，该方法必须在其子类得到实现，即产品的实例化被推迟到了子类；产品组件也有抽象产品和不同的实现类。

抽象工厂模式

抽象工厂与工厂方法不同在于工厂方法只能产生一个具体的产品对象，而抽象工厂的工厂组件是一个接口内有多个工厂方法，其实现类能够产生多个不同的产品对象。

Java注解

发表于 2017-04-26 | | 阅读次数

概述

Annotation可以看成是一个对Java元素的标记，这个标记不会影响原Java程序的执行，但一般会被AnnotationProcessor解析并以此来执行其他操作。比如最常见的@Override注解，它的AnnotationProcessor就是编译器，编译器会查看该方法对应类的父类是否有对应的方法，如果没有就会报错，告知程序员父类中没有被重写的方法。

注解的使用，Annotation是一个接口，但是我们在定义自己的注解时，不能直接使用这个Annotation去声明。而只能使用@interface去声明，其实这个@interface已经隐式继承了Annotation了。

基本注解

@Override、@Deprecated、@SuppressWarnings以及Java 7出现的@SafeVarargs和Java 8中出现的@FunctionalInterface是5个基本的注解。
@Override修饰方法，表示该方法必须是重写父类的方法，否则编译出错。
@Deprecated用于表示某个程序元素(类或者是方法)已过时，当其他程序使用该元素时会给出警告。

@SafeVarargs用于抑制堆污染相关的警告。
@FunctionalInterface仅用于修饰接口，表示该接口是一个函数式接口，如果不是则会编译出错。

常见的元注解

元注解是用来修饰注解的注解，最常见的有@Retention、@Target和@Inherit三个。其中@Retention表示该注解的保留时间，若成员变量值为RetentionPolicy.RUNTIME,则表示这个注解能在运行时通过反射机制获取。@Target表示这个注解可以修饰哪些程序元素。@Inherit修饰的注解，表示被该注解修饰过的类的子类会自动被该注解修饰。

Java集合

发表于 2017-04-26 | | 阅读次数

Java集合基础

Collection

Collection是一个集合接口，与Map接口不同的是，实现Collection的集合类中的元素都是value型的，而实现Map接口的集合类的元素是型的。

Set

无序、不重复是Set集合的典型特征，但是Java中为了更好的满足现实需要，使得Set的部分实现类并没有完全遵循这个特征。

HashSet

利用hash算法实现内存存储，完全满足Set两大特征，无序不重复。其中如何判断两个元素是否重复呢？HashSet中判断元素是否相等，是通过两个函数equals和hashCode，必须equals返回true且hashCode返回值相同才能判定元素相等，元素相同时不能对HashSet进行重复插入。所以，当要将某个类的对象放入HashSet时，不要轻易重写上述两个函数，如果需要，则一定要保证其一致性。
HashSet不是线程安全的，所以当不同线程操作同一个集合时，一定要注意保证同步，Java中的Collections中提供了线程安全的方法。

LinkedHashSet——HashSet的子类

与HashSet不同的是，LinkedHashSet使用链表维护元素的次序，即它打破了无序性这个特征。

TreeSet

TreeSet保证了元素的排序状态，即它使元素在集合中按照某种升序或者降序排列。HashSet是采用hash算法来决定元素的存储位置，而TreeSet则是利用红黑树的数据结构来存储数据。由于TreeSet要对元素进行排序，所以放入TreeSet的对象必须实现Compareable接口。注意：每当添加一个元素到TreeSet中时，会将该元素和集合中的其他元素进行比较如果compareTo(Object obj)返回的是0，则表示这两个元素是相等的无法插入。另外，重写equals方法时，应该保证equals和compareTo方法具有一致性。

各Set总结

当不需要保持一个集合中元素的排序状态时，最好使用HashSet，因为HashSet性能更好，他不要维持红黑树的结构。LinkedHashSet在插入、删除不如HashSet，但是遍历效率却高得多。

List

一种长度可变的动态数组，与set不同的是List实现类允许有序和重复元素。

ArrayList

由于List可以被看成是一个长度可变的数组，所以Java引入了capacity和length的概念。当没有指定数组长度的时候，会给该数组指定一个初始的capacity，只有当数组的实际长度大于capacity时，才对数组进行重新分配更大的空间。ArrayList的实现结构是数组，所以在索引方面效率高但是插入删除较低。遍历时采用随机访问要比使用iterator迭代器要好。

Queue

Queue接口用于模拟队列这种数据结构，满足FIFO。

PriorityQueue

作为Queue的实现类，每次poll或者peek获取的元素是队列中最小的元素，严格来讲已经违背了FIFO的队列特征。由于要对元素进行排序，因此元素对应的类必须实现Compareable接口。

Deque

Queue的子接口，双端队列，还可以被当成栈来使用。

LinkedList

不仅实现了List接口还实现了Deque接口，所以LinkedList可以被当成栈使用。LinkedList内部是链表的实现方式，在插入删除效率上比ArrayList要高。遍历时采用iterator迭代器要比随机访问要好。

Map

Map的集合实现类存放的元素是键值对类型，前面介绍到的Set可以看成是value为null的特殊Map，因此在一些子类的实现上具有相同性质。

HashMap

可以将value看成是key的附属，这样，HashMap和HashSet就可以看成同一种结构。判断两个元素是否一致，也是看HashMap中key的equals和hashCode方法。

LinkedHashMap

使用双向链表维护插入顺序和迭代顺序一致。

TreeMap

红黑树结构，保证Map中的元素按照key的大小保持一种排序状态，所以元素对应的类需要实现Compareable接口。
