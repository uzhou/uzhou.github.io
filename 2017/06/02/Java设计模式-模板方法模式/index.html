场景描述

还是吃饭问题：

场景1和场景2除了去餐厅方式不同和吃的东西不一样，其他的操作是一样的，这个时候可以将这些公共的操作提取到父类中，然后子类去实现这些不同的地方。

模式中的角色

抽象类：包含业务实现的逻辑顺序，即先做什么，再做什么。
子类：继承父类，并重写对应方法

钩子方法

从上述描述中，可以看出，这样的设计模式未免过于呆板，因为父类一旦确定，那么算法的流程就无法改变，这显然不太合理，所以提出了钩子方法，钩子方法一般默认是空实现，如果子类去实现了它，那么就相当于它改变了父类中算法的流程，增加了一个步骤。
另外，钩子方法也可以作为一个条件判断，来决定算法流程的走向。
