Java设计模式--外观模式

发表于 2017-06-02 | | 阅读次数

概述

通过引入一个外观角色来简化客户端与子系统之间的交互，为复杂的子系统调用提供一个统一的入口，降低子系统与客户端的耦合度，且客户端调用非常方便。

场景描述

自己在家吃饭和去餐厅吃饭两种场景：

上面的场景1描述了客户在家吃饭时要与众多的子系统交互，不如菜篮子买菜、电饭锅烧饭，餐具吃饭等，这就使客户在执行吃饭这个操作时与过多的子系统耦合；而场景2描述了客户去餐厅吃饭只需要与服务员说一声吃什么，然后这个服务员会与诸多的子系统交互完成功能，这就使客户与子系统解耦了。

模式中涉及的角色

client：功能的实际调用者
subsystem1、2、3…:子系统
外观：Facade
