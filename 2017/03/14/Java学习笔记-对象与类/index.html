对象与类

java中的思想就是万物皆对象，他是一个典型的OOP语言。OOP和面向过程编程的区别在于：OOP一般会先考虑数据结构，再去考虑算法；而面向过程编程则是先考虑算法，在考虑用什么数据结构。

对象很好的诠释了封装这个概念，而封装指的是将数据隐藏起来，只暴露部分接口给外部使用。为了不破坏这个封装性，一般会将类中的实例域定义为private型的，另外访问器方法不能返回可变对象的引用。例如：

final、static的实例域是有所区别的，其中final对象必须在构造器中赋予初值或者在此之前赋初值，且以后不可以改变；而static则表示这个实例域是属于这个类的，他会在类初始化时就被初始化。static还可以修饰method，表示这个方法不需要对象去调用直接使用类名就可以了。

初始化块：

  2. static初始化：类初始化时，执行。

包：包的主要作用是为了防止类的重名，另外还能够十分有序的对类进行管理；可以将包看成是一个类的集合。

private、protected、public的作用域：

  • private：当前类；
  • protected：当前类、子类和当前包内的其他类；
  • public：所有类；
