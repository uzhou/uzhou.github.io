前言

上篇博客简单介绍了，如何在MFC下实现图像拼接，但并未对图像拼接的过程中相关步骤进行显示。这也是本篇博客的目的，对图片特征值进行提取、显示和匹配。

实现

UI设计

为了方便，我们将上一篇博客的工程中的三张图片拼接，改为两张图片拼接，同时在对话框的设计上设置6个picture control。从1-6分别为图片1、图片2、图片1的特征值显示、图片2的特征值显示、特征值匹配结果图、最终拼接图。以及需要增加一个static text控件，用于显示相关步骤的时间消耗。具体实现效果图如下：

功能函数

ShowImageInPictureControl(Mat img, UINT ID,IplImage *KeyPointPic1)

该函数用于将img显示到控件为ID的picturecontrol中，keyPointPic1为存放该img的变量。

同样的，再使用keyPointPic1之前，要对其进行初始化。具体见上一篇博客的相关内容。同时该函数要在StichPictureDlg.h头文件中定义好。下面的函数也是相同的。

int CStichPictureDlg::StichProcess()

该函数，包含了所有特征值相关的操作和显示。

其中，KeyPointPic1和KeyPointPic2、OutImage都是添加的IplImage*变量。IDC_STATIC1则为UI中的static text的ID。
