Java注解

发表于 2017-04-26 | | 阅读次数

概述

Annotation可以看成是一个对Java元素的标记，这个标记不会影响原Java程序的执行，但一般会被AnnotationProcessor解析并以此来执行其他操作。比如最常见的@Override注解，它的AnnotationProcessor就是编译器，编译器会查看该方法对应类的父类是否有对应的方法，如果没有就会报错，告知程序员父类中没有被重写的方法。

注解的使用，Annotation是一个接口，但是我们在定义自己的注解时，不能直接使用这个Annotation去声明。而只能使用@interface去声明，其实这个@interface已经隐式继承了Annotation了。

基本注解

@Override、@Deprecated、@SuppressWarnings以及Java 7出现的@SafeVarargs和Java 8中出现的@FunctionalInterface是5个基本的注解。
@Override修饰方法，表示该方法必须是重写父类的方法，否则编译出错。
@Deprecated用于表示某个程序元素(类或者是方法)已过时，当其他程序使用该元素时会给出警告。

@SafeVarargs用于抑制堆污染相关的警告。
@FunctionalInterface仅用于修饰接口，表示该接口是一个函数式接口，如果不是则会编译出错。

常见的元注解

元注解是用来修饰注解的注解，最常见的有@Retention、@Target和@Inherit三个。其中@Retention表示该注解的保留时间，若成员变量值为RetentionPolicy.RUNTIME,则表示这个注解能在运行时通过反射机制获取。@Target表示这个注解可以修饰哪些程序元素。@Inherit修饰的注解，表示被该注解修饰过的类的子类会自动被该注解修饰。
