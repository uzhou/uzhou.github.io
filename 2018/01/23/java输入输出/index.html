java输入输出

发表于 2018-01-23 | | 阅读次数

IO

何为流，stream是指从源到目标的有序数据。java中以内存为中心，若内存到IO设备则是输出流，若从IO设备到内存则是输入流。

分类

输入和输出流

如上所述，从内存到IO则为输出，反之输入流。Java中，只能从输入流中取数据，不能放数据；只能向输出流中放数据不能从中取数据。

字节流和字符流

根据每次从流中取或者放数据的大小分：若每次操作一个字节的数据则是字节流，若每次操作一个字符即两个字节则是字符流。字符流一般用于文本数据的传输，而字节流则可以传输任何内容比如二进制数据。

节点流和处理流

可以直接从/向一个IO设备取/放数据的流称为字节流，一般根据不同的IO设备，节点流的处理是不一样的。处理流则是对节点流封装后的产物。这种设计模式是装饰模式，为了让上层应用在调用的时候更方便，消除不同节点流的差异。节点流的构造函数一般是以IO节点为参数，而处理流的构造函数则是以节点流为参数构造的。

处理流两个功能：

  • 性能提升：一般的节点流每次IO操作只能读取一个数据单元的数据，但处理流使用了缓冲区，并且OS层使用了DMA，cpu不用每次进行IO的系统调用都要“过问”一遍，极大提高了IO效率。
  • 操作方便：正是由于上述的变化，可以通过处理流一次读取一批的数据，而不是一个字节或者一个字符。

java1.4的NIO

上面的IO流，在读取不到数据时会让线程阻塞，不仅如此，底层的实现还是依赖于字节处理，面向流的输入输出每次只能处理一个字节的数据。

1.4NIO概述

NIO采用内存映射文件的方式来处理输入输出，NIO将文件或者文件中的一段映射到内存区域中，这样就可以像访问内存一样访问文件了。

channel

Channel和传统的InputStream、OutputStream最大的不同是Channel提供了一个map()方法，通过该方法可以将“一块数据”映射到内存中。另外Channel没有输入输出的概念，它是双向的，既可以取数据也可以放数据。

channel的使用

与传统流的两个区别：

  1. 直接将文件中内容映射成Buffer
  2. 程序不能直接访问channel中的数据包括数据的读取写入都不行，所有数据的操作都必须先经过Buffer

Buffer

Buffer的本质是一个数组，发送到Channel中的数据必须先放到这个Buffer中，从Channel中取数据时，Channel也必须先将数据放到Buffer中，然后从Buffer中取数据。

BUffer的使用

Buffer的四个概念：

  1. capacity：容量
  2. limit：位于limit后的区域既不能被读也不能被写
  3. position：用于指明下一个可以被读取或者写入的数据的索引位置。当使用Buffer从channel中读取数据，position表示已经读到了多少数据；当使用Buffer向channel中放数据时，则表示写了多少数据。
  4. mark：用于回滚的？

1.7NIO和1.4NIO

相比1.4的NIO来说，NIO2.0提供了基于异步channel的IO。可以说，1.4的NIO虽然是非阻塞的，但是仍然是同步非阻塞，到了1.7的NIO2.0才真正实现了异步非阻塞。
