IoC和DI

IoC的中文翻译是反转控制，貌似从这个字面上很难理解什么叫IoC。IoC具体指当一个对象创建时，它所依赖的对象由外部传递给它，而非自己去创建所依赖的对象（比如通过new操作）。因此，也可以说在对象如何获取它的依赖对象这件事情上，控制权反转了。DI是依赖注入的英文缩写，这其实是IoC的一种实现方式，也就是说DI是IoC的实现（IoC是接口，DI实现了IoC）。

Spring Aop原理及实现

发表于 2017-10-20 | | 阅读次数

Aop

Aop的全称是面向切面编程，这是相对于OOP来说的，OOP中是面向对象编程。AOP和OOP解决的都是代码复用问题，只不过OOP通过继承来解决纵向的代码复用，而AOP通过动态代理技术解决横向的代码复用。具体的例子：Person和Teacher都有name属性，Person是Teacher的父类，所以Teacher继承了Person后就会直接具有name属性这就是纵向的代码复用；而横向代码复用就比较复杂，比如说日志管理、性能检测这个业务逻辑在不同的场景中可能是一样的不需要改动的，但是为了代码复用我们不可能将其在每一个场景中都写一遍，所以此时需要代理技术将这些业务逻辑抽出来并织入到目标场景中。

Aop的实现基础

JDK动态代理技术和CGLib动态代理技术。Spring中的AOP底层也是使用这两个技术实现的，只不过根据不同的配置选择了不同的实现。以上两个技术可见上篇文章。

AOP术语

  • Jointpoint（连接点）：程序执行的位置。Spring中这个概念用的少，基本上都是对PointCut操作。连接点由两个信息确定：用方法描述的程序执行点；表示该执行点方位的位置信息（比如Before、after等）
  • PointCut：业务中，需要织入横向逻辑的程序执行位置，注意：Pointcut不具备表示方位信息的能力。由于大部分Aop都是针对于方法织入的，所以可以将Pointcut描述为一个方法的集合，该集合中的每一个方法都是需要织入横切逻辑的。
  • Advice：横向逻辑的具体实现，并包含了位置信息。比如前置增强、后置增强等
  • Target：需要增强的目标类
  • Weaving：将Advice添加到Pointcut的过程
  • Proxy：织入后产生的代理类
  • Aspect：Pointcut+Advice

代理

代理的意思就是代表某个对象去完成该对象的任务，不过，我们可以在这个代表对象上做点手脚，加上一些自己的东西。比如商品的代理商可以代表厂家销售商品，在销售商品的同时，代理商可以做其他的事，比如打广告、捆绑销售等。在java中，厂家是目标类（或者接口），销售是业务逻辑代码，打广告、捆绑销售是横向逻辑，代理商将业务逻辑代码和横向逻辑结合在了一起。
然后需要了解下，静态代理和动态代理的区别：静态代理指一个目标类（目标接口）对应一个代理类，且代理类在编译器就确定了；动态代理指在运行时才去生成代理类。

TreeMap介绍

TreeMap底层使用红黑树实现，而红黑树是一个接近平衡二叉查找树的树形结构。所以TreeMap是一个有序的集合，如果未指定比较器则默认使用自然排序对keys进行排序，否则使用指定的比较器对keys排序。
TreeMap中，基本操作的时间复杂度都在Log(n),不论是get、put还是remove。注意：对于任意一个有序的map，不论是否明确指定了比较器，该集合中的元素必须满足一致性相等的原则(即compare(a,b)==0和a.equals(b)返回相同的boolean值)。这个概念在Compareable和Comparator中提到了，虽然不满足该性质也不会报错，但会使有序集合的部分操作出现不符合预期的情况。
TreeMap是非线程安全的，所以要在多线程下使用集合，可以使用Collections#synchronizedSortedMap()包装下，或者使用并发包中提供的有序集合。

TreeMap的声明

和其他集合差不多，唯一的区别就是要实现NavigableMap。NavigableMap是一个SortedMap的子接口，主要提供了一些和有序集合相关的方法，比如floorEntry、lowerEntry等等。实现这个接口，表明TreeMap是一个有序的集合。

Comparable和Comparator介绍

发表于 2017-09-28 | | 阅读次数

Comparable介绍

Comparable接口会强制实现该接口的类实现一个compareTo(E e)方法，这个compareTo方法就是该实现类的自然排序方法。凡是实现了该接口的对象的集合都能够直接使用Arrays#sort或者Cllections#sort方法排序，另外实现该接口的对象还能作为有序集合的key，比如SortedMap的key或者作为SortedSet的元素。
什么叫自然排序，JDK中是natural ordering，不知道我翻译的准不准。对于一个Class C来说，natural ordering其实就是一致性相等的意思。所谓一致性相等，即C的两个对象e1和e2满足e1.compareTo(e2) == 0和e1.equals(e2)返回相同的boolean值。另外compareTo方法中的参数不允许为null，否则会抛出null异常。
JDK中强烈建议实现该接口的对象都要满足一致性相等的规则，这样可以保证那些未指定比较器，直接实现Comparable接口的有序集合不会表现出违背对应集合规范的行为。
例如：如果一个未指定比较器，直接实现Comparable接口的有序集合SortedSet,a、b为需要添加的两个元素且满足(!a.equals(b) && a.compareTo(b) == 0)，那么添加a之后，b是添加不进去的。

代码部分十分简单。

Comparator介绍

Comparators能够作为Arrays#sort方法或者Collections#sort方法的参数，用来指定对应集合应该使用什么样的比较器。Comparator是用来控制某两个元素应该如何确定大小关系的比较器，实现该接口并不一定能够使实现类拥有排序能力，只能说该实现类的对象是一个比较器。
Comparator也应该满足一致性相等的原则，这样指定该比较器的有序集合就不会出现奇怪的行为。

红黑树

发表于 2017-09-27 | | 阅读次数

红黑树定义

满足下面五条性质的二叉查找树称为红黑树：

  1. 每个节点要么为红色要么为黑色
  2. 根节点为黑色
  3. 每个叶子节点(NIL)为黑色
  4. 如果一个节点为红色，那么它的子节点一定为红色
  5. 从一个节点到该节点的子孙节点的所有路径上包含相同数目的黑色节点（该性质保证了红黑树是接近平衡二叉树的）

红黑树的应用

java中的TreeMap，Linux中的虚拟内存管理，JDK1.8中，HashMap也用到了红黑树。

红黑树的插入

首先红黑树是一棵二叉查找树，所以每次插入的位置一定是在叶节点处。而插入时能够保证它依然是二叉查找树，但是红黑树的性质不一定能满足。

  • 性质1很好满足，只需要涂色就可以；

HashMap源码介绍

HashMap底层使用哈希表实现，其实就是数组+链表的数据结构实现的，链表散列。具有以下特点：

  1. 非线程安全，要想使用线程安全的HashMap集合，可以使用Collections#synchronizedMap包装下，也可以直接使用HashTable和并发包中的集合。
  2. HashMap中元素的顺序可能会改变，这主要是扩容的时候对bucket索引的重新生成造成的。
  3. HashMap中的key不能重复，否则添加的时候会覆盖旧元素，允许null的存在。

    HashMap的继承依赖关系

HashMap继承了抽象类AbstractMap，而AbstractMap中提供了Map的一些公共操作。Cloneable和Seriablizable属于标记型接口，前者表示实现类可以使用clone方法而不抛出异常，但需要重写Object#clone方法；后者表示可序列化。

## HashMap主要field

首先介绍下几个概念：

  • capacity：the number of buckets in the hash table.HashMap中底层hash表的bucket数量
  • size：HashMap中元素个数
  • loadFactor：负载因子，用于描述哈希表在HashMap自动扩容之前能够达到多满的程度
  • threshold：当HashMap的size达到threshold时，会执行resize操作
  • modCount：表示集合结构修改的次数
    然后需要注意的一个地方就是为什么capacity必须是2的n次方：这主要涉及到位运算和取余运算之间的效率问题，后面会介绍到。
    通过这些属性我们知道，HashMap底层实现的数据结构就是数组和链表。

    HashMap构造函数

最重要的还是第一个构造函数，其他的构造函数最终会调用第一个函数。构造函数中为子类提供了一个init()钩子方法。

HashMap扩容相关

首先，我们需要知道在什么样的情况下会发生resize函数，添加元素的时候会改变size大小，当size>=threshold时，就会调用resize函数进行扩容。
resize主要根据newCapacity重新分配一个Entry数组，再讲table引用指向该数组；另外resize还调用了transfer函数将原Map中的元素转移到新分配的数组中，并对每一个元素调用indexFor方法，重新确定元素属于哪一个bucket。这就能解释为什么HashMap的元素顺序不是固定的。

上面两个函数完成了将Map中key转为table数组索引的步骤，首先回去出key的hashCode，并将这个hashCode传给一个扰乱函数，得到一个处理后的hash值，再将该hash值进行位运算得到该hash值对数组大小的取余，这个值就是该元素所在bucket的索引。
这里可以解释下为什么要进行扰乱：首先HashMap的效率受到元素是否均匀分布在table中，因此要想达到这种效果，一般是用元素的hashCode值对数组长度取余，但这样一来，对于那些hashCode低位有规律的元素来说会造成不好的分布。此时我们需要将hashCOde的高位特征和低位特征都利用起来这样的话就能减少碰撞。所以出现了扰乱函数：先对原hashCode移位，再将高位和低位进行异或运算，这样就变相利用了高位低位特征。
然后我们说明一下为什么capacity一定是2的n次方：indexFor函数中的h & (length-1) 和 h%length 等价，但是前者属于位运算其效率远大于后者的取余运算。上述等价的条件就是length必须是2的n次方。

HashMap的基本操作

put方法是借助于addEntry()方法完成的，在addEntry中我们可以看到if ((size >= threshold) && (null != table[bucketIndex]))的判断，这就是添加元素时要进行判断是否要扩容。

get方法就比较简单，不可能考虑扩容问题，首先是根据key获取扰乱后的hash值，然后由indexFor()得到索引值，最后在这个bucket下遍历扎到key一样的value并返回，找不到则返回null.

类的介绍

LinkedList底层基于双向列表实现，这与ArrayList不同，所以他们的特点很大程度上就是数组和链表的区别。比如LinkedList在添加和删除的性能上要优于ArrayList，但是在访问元素的性能上要劣于ArrayList。LinkedList是非线程安全的，类中提供的迭代器iterator和listIterator都满足fail-fast机制。

List介绍

List继承了Collection接口，这就决定了List是一个存放元素的集合接口。与Set不同的是，List的实现类可以根据索引去访问集合元素，另外List中允许有重复元素的重现，允许多个null，List会维护一个固定的元素顺序。

ArrayList源码分析

ArrayList的声明

首先继承了AbstractList，打开AbstractList可以看到这里面包含的是一些List集合共有的方法，比如迭代器、clear等；另外像add、set等需要具体实现的方法则需要子类去重写。
再者，继承List接口表明这是List的实现类；RandomAccess按照源码中的注释这是一个标记型的接口，表明ArrayList是可以随机访问的，RandomAccess中提到实现该接口的集合，使用for循环遍历要比迭代器遍历更快。
Cloneable，也是一个标记型接口，实现了该接口就可以使用Object中的clone方法且不会报CloneNotSupportedException，实现此接口的类需要重写clone方法，但注意这个clone方法不是Cloneable提供的。
Serializable与序列化有关的标记型接口。

ArrayList的属性

serialVersionUID一个与序列化有关的filed，默认为1L，反序列的时候会将传过来的serialVersionUID的值与本地实体中的serialVersionUID比对，如果一致则可以进行反序列化，否则会出现版本不一致的异常。
DEFAULT_CAPACITY：由于ArrayList基于数组实现的，且它不像Array一样，大小是固定的，也就是说ArrayList的大小可以改变，而DEFAULT_CAPACITY就是这个底层实现的数组的大小；与size区分开来：size是ArrayList中元素的个数；默认容量为10.
Object[] elementData：底层实现的数据，注意它的关键字是transient，表示这个field不会参与序列化。上述的Capacity就是这个数组的大小。

父类AbstractList中声明的，当类的结构发生改变时，这个值就会+1；一般用于迭代器中，当迭代器在遍历集合时，如果发现modCount和开始时的值不一样，那么就可以认为该集合在迭代器遍历的时候，结构发生了变化此时抛出ConcurrentModificationException异常。

HashSet

HashSet是一个存储不重复元素的集合，其底层实现是HashMap。HashSet有两个主要的field：HashMap m 和final Object PRESENT。其中m是实现该HashSet对应的HashMap，PRESENT则是HashMap中的value值，即m中的key集就是HashSet的element，而m中的value则永远为PRESENT。
m中有三个属性十分重要：size（表示集合中实际元素的个数），capacity（表示实现HashMap的哈希表大小），load_factor（表示该哈希表的负载因子）。一般的构造HashSet时，会默认HashSet的capacity为16，load_factor为0.75。
值得注意的是HashSet是线程不安全的，因此如果想多个线程下对HashSet访问不出错，必须有Collections#synchronizedSet方法包装下HashSet才能得到一个线程安全的集合。
Java集合采用fail-fast检错机制，一旦A线程访问HashSet时，有B线程改变了HashSet的结构（使用add、remove等方法），那么A会抛出异常。这个机制不能完全保证能够检测到并发问题，也就是说你不能依靠这个机制去完全避免并发问题。
HashSet在序列化的时候，不需要对m序列化，因为定义m的时候用了transient关键字，该关键字表示该变量不需要序列化。所谓序列化就是对象的属性传输到缓存或者存储区，以便日后能够根据对象的属性还原对象。

SortedSet

上面提到传统意义上的Set都是添加无序的，而实现了SortedSet接口的则可以实现添加有序。因为在SortedSet中的元素必须都实现同一个Comparator接口，该接口用于对集合中的元素进行排序，当未指定Comparator时，会采用默认的自然排序。

TreeSet

TreeSet基于NavigableMap实现的，就像HashSet基于HashMap实现的一样。不同的是，TreeSet是元素有序的集合，并且执行add方法时如果被添加的元素和当前集合的Comparator的接口不一样，那么会抛出类型转换的异常。
